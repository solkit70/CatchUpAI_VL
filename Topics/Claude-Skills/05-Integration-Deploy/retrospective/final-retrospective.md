| 라이브러리 | 용도 | 숙련도 |
|------------|------|--------|
| youtube-transcript-api | 자막 추출 | ⭐⭐⭐⭐ |

| pydub | 오디오 분석 | ⭐⭐⭐⭐ |

| ffmpeg-python | 영상 편집 | ⭐⭐⭐ |

### 3.3 문제 해결 경험

1. **Windows UTF-8 인코딩**
   - 문제: 한글 출력 시 UnicodeEncodeError
   - 해결: `io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')`

2. **YouTube Transcript API 변경**
   - 문제: 기존 API 인터페이스 변경
   - 해결: 새 API 패턴 학습 및 적용

3. **FFmpeg 필터 체인**
   - 문제: 복잡한 영상 구간 편집
   - 해결: filter_complex로 구간 추출 및 병합

## 6. 핵심 인사이트

### 6.1 기술적 인사이트

1. **"Skill은 프롬프트 엔지니어링의 결정체"**
   - 잘 설계된 SKILL.md 하나가 복잡한 워크플로우를 자동화

2. **"API 버전 관리의 중요성"**
   - 외부 API는 언제든 변경될 수 있음
   - 에러 핸들링과 대안 로직 필수

3. **"작은 도구들의 조합"**
   - pydub + Whisper + FFmpeg의 조합으로 강력한 편집 도구 구현
   - 각 도구의 강점을 활용

### 6.2 학습 인사이트

1. **"실패는 최고의 학습"**
   - M3의 API 오류 해결 과정에서 가장 많이 배움
   - 문제 해결 경험이 실력을 키움

2. **"문서화는 미래의 나를 위한 선물"**
   - WorkLog와 Retrospective가 나중에 큰 도움
   - 잘 정리된 문서가 재사용 가능한 자산

3. **"완벽보다 완료"**
   - 100% 완벽한 코드보다 작동하는 MVP가 중요
   - 이후 점진적 개선 가능
